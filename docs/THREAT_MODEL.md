# 首版威胁模型

## 1. 保护对象

- 文件内容、文件名、目录关系和缩略图；
- 个人身份、设备密钥、恢复材料和会话；
- 分享对象、联系人和访问历史；
- 元数据数据库、备份清单和审计记录；
- 服务可用性及用户恢复数据的能力。

## 2. 主要攻击者

- 扫描公网服务的自动化攻击者；
- 尝试暴力破解或盗用分享链接的人；
- 家庭网络中的恶意或已感染设备；
- 存在漏洞或恶意行为的客户端和应用；
- 能够盗走硬盘、备份介质或整台服务器的人；
- 被控制的中继、DNS、云服务或更新分发节点；
- 利用依赖、构建和升级流程的供应链攻击者。

## 3. 安全目标

- 公网没有可暴力破解的万能管理员密码；
- 设备、会话和分享凭证具有不同作用域并可撤销；
- 盗取关机服务器或备份介质不能直接读取数据；
- 单个解析器、缩略图任务或应用被攻破后不能控制全部节点；
- 删除、勒索和数据库损坏后仍存在可验证恢复路径；
- DNS或中继被控制时不能冒充已固定的长期身份；
- 关键行为可被用户理解和审计。

## 4. 非目标与现实限制

- 不承诺抵御拥有无限资源的攻击者；
- 不承诺运行中节点被完全控制后明文仍绝对不可得；
- 不把杀毒扫描视为阻止所有恶意文件的保证；
- 不保证首版抵抗所有流量分析和元数据观察；
- 不以自托管为由把配置错误责任全部推给用户。

## 5. 必要控制

### 身份与管理

- 首次管理使用本地或物理邻近配对；
- 管理员设备使用硬件支持的密钥保存能力；
- 高风险操作进行再确认并留下记录；
- 恢复流程不能仅依赖正在恢复的节点；
- 提供设备列表、最后活动时间和一键撤销。

### 网络

- 默认只暴露一个经过加固的协议入口；
- 管理界面与数据面进行权限隔离；
- 连接、上传、分享和认证均有限速与配额；
- 不信任客户端提供的文件名、路径、类型、摘要和长度；
- 中继仅转发端到端加密内容。

### 存储与处理

- 数据块、元数据和备份分别进行完整性验证；
- 文件解析、预览和媒体转码在低权限沙箱运行；
- 上传隔离区与正式数据区分离；
- 日志不记录密钥、完整分享令牌或敏感正文；
- 根密钥、Vault密钥和应用能力分层隔离。

### 更新与恢复

- 发布物和更新清单需要签名；
- 更新应可原子应用并自动回滚；
- 依赖和构建来源可追踪；
- 定期验证备份而不是只报告“上传成功”；
- 提供安全模式和从可信介质重建流程。

## 6. 发布门槛

在称为Beta版本之前至少完成：

- 公开威胁模型评审；
- 认证握手和密钥生命周期评审；
- 文件上传、分享与解析器模糊测试；
- 权限绕过和跨Vault隔离测试；
- 断电、磁盘满、数据库损坏和备份恢复演练；
- 外部安全审计及修复验证。
