# 手机备份边界与方案

## 1. 可以可靠实现的内容

### Android

- 用户授权的照片和视频；
- 通过系统文件选择器授权的目录和文件；
- PSN客户端自身的数据；
- 主动集成PSN SDK并允许导出的第三方应用数据。

### iOS

- 经PhotoKit授权的照片和视频；
- 经文件选择器或文件提供器访问的用户文件；
- PSN客户端自身的数据；
- 主动集成导出或同步接口的第三方应用数据。

## 2. 不能通用承诺的内容

普通第三方备份应用不能任意读取其他应用的私有数据库、登录令牌、缓存或内部文件。Android 10及以上默认使用分区存储，应用不能访问其他应用专属目录；iOS应用通常被限制在自己的沙箱内。

即便平台自身提供应用备份，应用开发者也可以控制包含范围，部分数据因安全或可重建原因被排除。因此“完整恢复所有应用到原状态”需要操作系统厂商、设备管理能力或每个应用主动配合。

## 3. 产品分阶段方案

### 第一阶段：媒体和文件

- 相机照片自动备份；
- 截图、视频和选定相册；
- 用户选择的下载及文档；
- Wi-Fi、充电、漫游和电量策略；
- 原图校验、重复检测和删除保护。

### 第二阶段：标准系统数据

经独立授权同步联系人、日历等平台提供标准API的数据。每种数据类型单独建模权限和冲突，不能因为用户允许照片就同时读取联系人。

### 第三阶段：应用数据SDK

为第三方应用提供：

- 应用专属Vault空间；
- 结构化记录和文件对象API；
- 配额与版本；
- 设备间同步；
- 导出、删除和迁移；
- 由用户批准的数据共享能力。

应用主动接入后，用户可以备份和迁移其业务数据，但认证令牌、反作弊状态等安全敏感内容仍应由应用决定是否可恢复。

### 高级实验：系统级备份

Root、越狱、ADB、设备厂商接口或企业设备管理可能扩大访问范围，但会带来兼容、安全、商店政策和用户支持成本。它们不能成为大众版本依赖，也不得弱化设备本身的安全机制。

## 4. 照片备份的删除规则

首版不应实现“服务器上传成功后自动删除手机原图”。正确顺序是：

1. 上传原始内容和必要元数据；
2. 服务端完成分块与完整文件校验；
3. 数据进入独立备份并通过策略检查；
4. 客户端向用户显示可释放空间；
5. 删除必须由用户明确确认，并遵循平台照片权限。

同步删除和备份保留必须是两个开关，避免手机误删传播到所有副本。
