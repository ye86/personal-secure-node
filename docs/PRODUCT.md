# PSN Drive 产品定义

## 1. 产品一句话

PSN Drive 是运行在用户自己服务器上的个人网盘：自动备份电脑和手机中的照片与文件，通过个人域名安全访问、同步和分享，并让用户清楚知道数据存在哪里、占用多少空间、被谁访问过。

首批用户是拥有迷你主机、旧电脑、NAS或自建服务器经验的DIY用户。第一版优先提供可靠、透明、可恢复的文件服务，不以替代所有公有云功能为目标。

## 2. 核心价值

- 数据保存在用户控制的设备中；
- Windows、Android、iOS和浏览器可以访问同一份数据；
- 手机照片和用户文件自动备份；
- 文件只保存一份，目录、相册和应用通过引用组织；
- 分享可设置接收者、有效期、次数和容量；
- 用户可以检查空间、版本、设备、授权和访问记录；
- 更换客户端或应用不会丢失底层数据。

## 3. 首版用户任务

### 3.1 初始化节点

用户在家庭服务器安装PSN Drive，创建管理员设备和恢复材料。系统完成磁盘检测、数据目录初始化、HTTPS入口和基础健康检查。

### 3.2 绑定设备

Windows或手机客户端扫描一次性二维码，双方核对节点身份后登记设备密钥。设备丢失时可由其他管理员设备撤销。

### 3.3 自动备份

- Windows：同步用户选择的文件夹；
- Android：备份经用户授权的照片、视频及用户选择的目录；
- iOS：备份经系统照片库或文件选择器授权的内容；
- 浏览器：手动上传、下载和管理文件。

### 3.4 查看与整理

用户可以按目录、时间、类型、设备和标签查看数据，并看到逻辑大小、实际占用、历史版本和备份状态。

### 3.5 分享与收件箱

用户可以向指定PSN身份发送文件，或创建限时浏览器链接。外部上传首先进入隔离收件箱，通过容量、文件数量、有效期和速率限制后再由用户归档。

### 3.6 恢复

用户可以恢复误删文件、历史版本、单个设备目录或完整节点。备份是否真实可恢复必须由定期恢复演练验证。

## 4. 手机应用数据范围

“应用数据”必须分级表达，避免让用户误解。

| 等级 | 数据 | 首版支持方式 |
|---|---|---|
| A | 照片、视频、下载和用户文档 | 系统授权后直接备份 |
| B | 联系人、日历等标准数据 | 经系统专用API和单独授权，后续提供 |
| C | 支持PSN导出/同步接口的应用数据 | 由应用主动写入其PSN空间 |
| D | 其他应用的私有数据库、令牌和内部文件 | 普通客户端无法通用读取，不承诺支持 |
| E | Root、越狱或系统级备份 | 仅作为高级实验，不作为默认产品路径 |

Android的分区存储会阻止普通应用访问其他应用的专属目录；应用自己的备份内容也可由开发者排除。iOS应用通常只能访问自己的沙箱，照片和文件需要通过平台授权接口获取。因此首版宣传应使用“照片和文件备份”，而不是“完整手机镜像备份”。

## 5. 首版功能范围

### 必须完成

- 单用户节点和管理员设备；
- Windows文件夹同步；
- Android照片及选定文件备份；
- 响应式Web管理界面；
- 文件上传、下载、移动、复制和删除；
- 分块上传、断点续传和完整性校验；
- 文件版本、回收站和快照；
- 存储配额与空间账单；
- 设备绑定、短期会话和撤销；
- 限时下载链接与隔离上传链接；
- 加密传输、静态加密和加密备份；
- 更新、日志、健康检查和恢复说明。

### 可以稍后加入

- iOS原生客户端；
- 多用户家庭空间；
- 相册、人脸或内容识别；
- 节点到节点直传；
- NAT穿透和不可信中继；
- 第三方应用数据SDK；
- 本地全文搜索和AI整理。

### 首版不做

- 通用聊天和视频通话；
- Windows应用兼容舱；
- 任意第三方应用私有数据抓取；
- 社交网络、应用商店或物联网平台；
- 公网匿名注册；
- 无证据的“绝对安全”承诺。

## 6. 成功指标

- 新用户在30分钟内完成安装和首台设备备份；
- 网络中断后可自动续传且不产生损坏文件；
- 同一文件重复上传不会重复占用完整空间；
- 从备份恢复随机抽取文件的校验结果一致；
- 丢失设备被撤销后无法建立新会话；
- 用户能够解释每类空间占用和每个外部分享；
- 10名真实DIY用户连续使用至少30天，并将其作为日常文件备份之一。

## 7. 产品边界

PSN Drive首先是个人控制的数据副本，不应鼓励用户把首版作为唯一副本。正式发布前至少遵循“工作副本、节点副本、独立备份”三份数据原则，并明确测试硬盘损坏、断电、数据库损坏、误删和密钥丢失场景。
