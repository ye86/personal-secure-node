# 数字主体

这里收录关于“数字主体”设想的公开文稿。

这个设想从一个朴素的问题出发：当恶意软件进入个人电脑，用户为什么往往既无法还原它从哪里来，也无法立即制止它继续修改系统、读取数据和向外通信？如果计算机是一种属于人的工具，它是否也应该拥有类似刹车、边界、授权记录和责任线索的基础能力？

- [完整稿：数字主体——互联网控制、授权与责任的候选框架](./manuscript-v0.2/DIGITAL_SUBJECT_MANUSCRIPT_V0.2.md)

  更完整地展开概念的产生顺序、系统与应用边界、个人数据、可信调用、物联网和实现路径。

## 关于这组文稿

“数字主体”在这里不是一个已经完成的标准，也不是对现有互联网的整体替代方案。它是一种候选出发点：让设备、应用和网络行为能够表达“谁在行动、依据什么授权、越过了什么边界、事后如何追溯”，并据此讨论机器怎样更可靠地听从于人。

文稿保留问题、取舍和尚未解决的部分，欢迎围绕其收益、代价、技术可行性与可能产生的新风险展开讨论。

文稿在整理过程中使用了人工智能辅助结构梳理、语言修改、反方审视和引用核验。核心问题、概念推导与观点取舍由作者提出并确认，作者对公开内容负责。
