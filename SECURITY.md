# 安全策略

## 当前安全状态

本仓库是研究原型，尚未经过独立安全审计，不能用于保护真实个人数据、生产密钥或高价值账户，也不应直接暴露到公网。

当前实现会把 Ed25519 私钥以未加密 PEM 文件写入工作目录；共享秘密也尚未经过正式 KDF 和完整握手协议处理。这些行为仅用于概念演示。

## 报告漏洞

请优先使用 GitHub 仓库的 **Private vulnerability reporting / Security advisory** 功能报告漏洞。不要在公开 Issue、讨论区或社交媒体中发布可利用细节、真实密钥或个人数据。

报告中建议包含：

- 受影响的文件、版本或提交；
- 复现条件和最小复现步骤；
- 可能造成的身份、数据或可用性影响；
- 已知缓解方式；
- 在不泄露敏感信息前提下提供的日志。

在维护者确认并提供修复时间表之前，请避免公开披露利用细节。

## 密钥泄露

任何曾进入公开仓库、构建日志、Issue或聊天记录的私钥都应视为已经泄露，删除文件不能恢复其可信性。应生成新密钥、撤销旧身份绑定，并检查历史使用记录。

仓库根目录的 `.gitignore` 会忽略常见本地密钥和运行产物，但忽略规则不能移除已经被 Git 跟踪或写入历史的文件。

## 安全设计原则

- 不自行设计未经分析的密码算法；
- 默认拒绝，按用途提供最小能力；
- 根身份不交给应用或远程服务；
- 网络位置不等于信任；
- 失败时限制损失并保留恢复路径；
- 安全声明必须有测试、威胁模型或审计证据支持。
